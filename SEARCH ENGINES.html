Search Engine Optimization (SEO)
There are two places where SEO is implemented. First in websites and second in keyword advertising.  Here I will a discuss the main points regarding SEO and Websites.

Preparation
There is little reason to spend time and money optimizing a website
for search engines if that web site is not ready. Visitors will not stay if your
website is not:
  • Aesthetically appealing
  • Focused
  • Easy to use
  • Mobile friendly

All of these points reduce credibility, increase frustration and increase the likelihood of a visitor moving on.

Search-ability
Currently there are two key search engines in North America: Google and Bing.
In order for these search engines to be aware of you site it must be indexed. A web site
cannot be indexed if it contains a robots.txt file which contains directives to ignore
a site. For details see https://support.google.com/webmasters/answer/6062608?hl=en

Keywords
Keywords are the words and terms typed into a search engine. A web site should be designed
to match keyword searches by including words and terms that a search engine can translate into a direct match.
The goal of a search engine is to abstract web content into terms that accurately summarize that nature of you website, so that it can present authentic and useful results.

What search engines use
Search engines can be told about keywords and given descriptions-but they do their own thinking now.  The "keywords" "meta tag and the "description" meta tag are still used, but they are largely de-emphasised in favor of actual website content and website name.   In a sense, search engines define your keywords for you.

Credibility is a key issue in how search engines rank results. The most important factor in credibility is backlinks. In other words
the number of other websites that point to your website.  

Search ranking will be negatively impacted by:
  • Not using SSL/TSL
  • Adult content
  • A lack of focus
  • Too many ads

How to structure content for SEO
Search engines parse and analyze content.  This means the domain name should be a clue to site content. 
In fact, interior URLs should also be used to impart information related to search terms.  Next comes the site title which should be  highly supportive of (and consistent with) site content. Title-casing is recommended for readability.  In order to make it physically visible,  the title length should be  short as possible but still informative.  I would say no more that 30 characters.

Search engines factor site structure and hierarchy to further quantify content. For this reason, placing an important term in an h1 tag is better than placing it in a div tag with a class to emphasize it.  For the same reason, text with a larger font or placed nearer the top of a site is given more weight in evaluating the purpose of a site.

Conversely, images and embedded content such as video, do not support a sites theme because they are not read or interpreted by search engines.   At best, a search engine can using image and video file names to support a sites theme.

Further optimization
There are to additional things that will increase the attractiveness of a web site. First, include a sitemap that can be understood by search engines.

Second,  increase site comprehension by search engines using standardized metadata.  There are several vocabularies available to do this.  I discovered schema.org first, and for this reason have a bias towards it.  Schema.org is a community effort to define a common semantic vocabulary used primarily for annotating websites.  Annotations, are implemented as small key=value attribute pairs which impart relationships and content type in html using a non-invasive form. For example,
<section  itemtype='http://schema.org/PostalAddress'>
Street: <span itemprop="streetAddress">123 Somelane</span>
City: <span itemprop="city">Windy Butte</span>
State: <span itemprop="state">Windy Butte</span>
